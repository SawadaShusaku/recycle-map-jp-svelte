# Note欄 データ調査レポート

調査日: 2026-05-19
対象DB: Production D1 (RECYCLING_DB)

## 1. 概要

`facilities.notes` および `place_collection_entries.notes` において、施設種別名や営業時間表現が大量に混入していることが判明した。  
notes非Nullレコードの **95%以上** が、これらの不要な文字列である。

## 2. 調査対象レコード数

| テーブル | 総レコード数 | notes非Null数 |
|---|---|---|
| facilities | 25,463 | 6,267 |
| place_collection_entries | 31,190 | 6,578 |

## 3. 単独ワード（分類ラベル）の混入状況

notesに入っている短い文字列（6文字以下）を集計したところ、以下の単独ワードが両テーブルでほぼ同じ比率で出現した。

| notes（単独ワード） | facilities | entries | 性质 |
|---|---|---|---|
| 郵便局 | 3,582 | 3,601 | 施設種別 |
| 自治体施設 | 2,033 | 2,262 | 施設種別 |
| 図書館 | 143 | 167 | 施設種別 |
| お店 | 115 | 138 | 施設種別 |
| 民間施設 | 61 | 71 | 施設種別 |
| その他 | 27 | 28 | 分類ラベル |
| 常設 | 1 | 1 | 分類ラベル |
| **合計** | **5,962** | **6,268** | — |
| **notes非Nullに対する比率** | **95.1%** | **95.3%** | — |

### 3.1 特記事項
- **すべて完全一致**: 上記の単独ワードは、いずれも `notes = '郵便局'` のように、その文字列**だけ**がnotesに入っており、他のテキストとの混在はない。
- **両テーブル同時混入**: facilities.notes と place_collection_entries.notes の両方に、ほぼ同じ件数で同じ単語が入っている。これはデータ収集パイプラインで、facilityとそのcollection entryの両方に同じnotesが付与されたことを示唆する。

## 4. その他のnotes（残り5%）の内容

単独ワードを除いた残りのnotesを集計したところ、上位はすべて**営業時間・回収日時**に関する表現であった。

### 上位例
| notes | facilities | entries |
|---|---|---|
| 毎月第2・第4土曜日の、午前10時から正午まで | 31 | 31 |
| 回収員手渡し方式による回収（毎月第2・4土曜日の午前10時～正午） | 24 | 24 |
| 毎月第3土曜日（午前9時から正午まで） | 20 | 21 |
| 毎月第2土曜日（午前9時から正午まで） | 20 | 21 |
| 毎月第4月曜日（祝日の場合は、翌日の火曜日）午前9時から午後0時30分まで | 20 | 20 |
| 毎週水曜日（祝日・年末年始を除く）の11時から15時まで | 18 | 18 |
| 第４木曜日午前９時～正午 | 17 | 17 |
| 午前9時から午後5時 | 8 | 8 |
| … | … | … |

### 4.1 特記事項
- これらも **両テーブルにほぼ同じ内容**が入っている。
- 「開館時間」という文字列そのものは少ないが、「午前9時から午後5時」のような**時間帯表現**が大量にある。ユーザーが指摘していた「開館時間など」は、これらの時間表現の混入を指していたと考えられる。

## 5. まとめ・判断材料

### 削除対象として明らかなもの（分類ラベル）
以下は明らかに施設種別やデータソースの分類タグであり、ユーザー向けNoteとして表示すべきでない。

- `郵便局`
- `自治体施設`
- `図書館`
- `お店`
- `民間施設`
- `その他`
- `常設`

→ **合計約6,000件（notes非Nullの95%）**

### 要検討もの（営業時間・回収日時表現）
残りの約5%は、営業時間や回収日時に関する具体的な情報である。これらは：
- **hoursフィールドとの重複の可能性**: 同じ内容が `hours` に入っているか確認が必要
- **有用性**: 回収日時（「毎月第2土曜日」など）はユーザーにとって有用な情報の可能性がある
- **表示場所**: 現在はnotes（基本情報タブ）に表示されているが、hours欄や別のUI要素に統合すべきか検討が必要

## 6. 推奨対応方針

### 即座に実装可能（APIサニタイズ層）
分類ラベル（郵便局、自治体施設、図書館、お店、民間施設、その他、常設）を `sanitizePublicText` で null に変換。影響範囲が限定され、安全に検証可能。

### 要ユーザー判断
営業時間・回収日時表現については、以下の判断をいただきたい：
1. **そのまま表示するか**: notesとして表示を維持（ただしhoursとの重複に注意）
2. **hoursフィールドに統合するか**: データパイプライン側で `hours` に移動
3. **APIサニタイズで除外するか**: 時間表現を含むnotesを一律で非表示にする

### 根本対応（データパイプラインフィードバック）
- データ収集時に分類ラベルが `notes` に入らないよう、パイプライン側の修正を依頼
- `hours` とは別に `collection_schedule`（回収日時）のような専用フィールドを設ける検討
