## Context

現在のサイドバー（`SettingsSidebar.svelte`）は「表示設定」セクションを上部に、「情報・ヘルプ」セクションを下部に配置している。情報・ヘルプの4項目のうち「使い方」のみがリンク付きで有効化されており、「データについて」「更新情報」「プライバシーポリシー」の3ページは対応ルートが存在せず `disabled` 状態になっている。

`/static/decorations/` に14種類のSVGデコレーション素材が存在し、`src/lib/components/decorations/` にはSvelteコンポーネント版（Blob, Dots, Scribble, Sparkle）が整備されているが、既存のコンテンツページ（usage）でこれらが十分に活用されていない。

## Goals / Non-Goals

**Goals:**
- サイドバーのセクション順序を「情報・ヘルプ → 上部」「表示設定 → 下部」に変更する
- `/about-data`、`/updates`、`/privacy-policy` の3ページを新規実装し、サイドバーからナビゲート可能にする
- `/static/decorations/*.svg` アセットを新規ページおよび既存の `/usage` ページのビジュアル装飾として統合する
- 各新規ページはREADMEのデータソース情報・利用規約を元にコンテンツを記述する

**Non-Goals:**
- サイドバーの根本的なナビゲーション構造の変更（ドリルダウン2段構造は維持）
- データについてページでの動的データ取得（静的コンテンツのみ）
- 更新情報の自動生成・CMSとの連携
- 多言語対応

## Decisions

### 決定1: 新規ページはSvelteKit静的ルートとして実装

**選択**: `src/routes/<name>/+page.svelte` として静的コンテンツページを作成。  
**理由**: データソース・プライバシーポリシー・更新情報はすべて静的テキストコンテンツであり、D1クエリやAPIが不要。SvelteKitのSSRでビルド時にHTMLが生成されるため、Cloudflare Workers静的アセットとして高速配信できる。  
**代替案**: `/about-data` をAPI経由でデータソース情報を取得する動的ページにすることも考えられたが、データソース情報は頻繁に変わらないため静的が適切。

### 決定2: デコレーションはSVGファイルを `<img>` タグで参照

**選択**: `/static/decorations/*.svg` を `<img src="/decorations/blob_green_blue.svg">` として参照し、既存の `Blob.svelte` などコンポーネントとの共存を可とする。  
**理由**: コンテンツページでは複雑なSvelteコンポーネントより素のSVG参照が軽量で、ページ全体のデザインを迅速に組み込める。アニメーションが必要な場合はCSSで制御可能。  
**代替案**: `src/lib/components/decorations/` の既存コンポーネントをインポートする方式も選べるが、既存コンポーネントがSvelte 4スタイル（`export let`）で書かれているため、Svelte 5 runesモードとの整合性をコンポーネントごとに確認する手間が生じる。

### 決定3: サイドバーの順序変更はテンプレートのみで完結

**選択**: `SettingsSidebar.svelte` のテンプレート内で `settingSections`（表示設定）と `infoMenuItems`（情報・ヘルプ）の描画順を入れ替えるだけとし、データ構造・スタイルは変更しない。  
**理由**: UIの順序変更は純粋なレイアウト変更であり、最小変更でユーザー要求を達成できる。

### 決定4: プライバシーポリシーのスコープ

**含める内容**:
- Geolocation API の利用（経路検索・現在地取得）
- localStorage の利用（マーカースタイル・フォント設定の保存）
- Mapillary API へのリクエスト（施設周辺画像取得）
- 地図タイルサービス（OpenFreeMap）へのリクエスト
- ルーティングAPI（OSRM）へのリクエスト
- アクセス解析・クッキーの不使用を明記

**含めない内容**: 個人情報の収集（アプリがユーザーデータをサーバーに送信しないことを明記）

## Risks / Trade-offs

- **更新情報の陳腐化リスク** → `更新情報` ページは定期的な手動更新が必要。初回は直近のメジャー変更（全国対応、D1移行等）を記載し、将来の更新フローはコメントで案内する。
- **デコレーション素材の視認性** → SVGをそのままページ背景・装飾に使う際、テキストコントラストが低下する可能性。opacityを調整し、重要テキストとの重複を避けるレイアウトで対応。
- **使い方ページのデザイン変更がユーザーの学習フローを壊すリスク** → 基本的な情報構造（セクション一覧・注意事項・各ステップ）を維持したままビジュアルのみ刷新するため影響は最小限。

## Migration Plan

1. `SettingsSidebar.svelte` のセクション順序変更（1行の変更）
2. 新規ページ3件（`/about-data`, `/updates`, `/privacy-policy`）の作成
3. `SettingsSidebar.svelte` の `infoMenuItems` で3ページのhrefを有効化
4. `/usage` ページのビジュアルにデコレーション素材を追加
5. `npm run smoke` でSSR確認
6. E2Eテスト（リンク遷移・ページ表示）
